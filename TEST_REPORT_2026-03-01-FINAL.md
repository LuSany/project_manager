## 🔧 已修复问题详情

### P1 级别问题 ✅ 已完成

#### Risk API 测试失败 (16 个失败 → 0 个失败)

**文件**: `tests/unit/risk.test.ts`

**问题根因**:
测试期望与实际路由实现严重不匹配，具体表现为:

1. 数据类型错误：测试使用字符串枚举，实现使用数字
2. 默认值不匹配：测试期望 'OPEN'，实现使用 'IDENTIFIED'
3. Include 对象错误：测试期望 task，实现使用 owner
4. 缺失字段：测试数据缺少 category, riskLevel, ownerId, progress
5. 导入缺失：GET/DELETE 函数导入不完整
6. Mock 不完整：缺少 prisma.project.findUnique mock

**修复方案**:

##### 1. 更新数据类型 (影响 8 个测试)

```typescript
// 修复前
probability: 'HIGH' as const,
impact: 'HIGH' as const,
status: 'OPEN' as const,

// 修复后
probability: 5,  // 数字 1-5
impact: 5,       // 数字 1-5
status: 'IDENTIFIED' as const,  // Schema 默认值
```

**数据类型映射表**:
| 字符串枚举 | 数字值 | 使用场景 |
|------------|--------|----------|
| 'HIGH' | 5 | 高概率/高影响 |
| 'MEDIUM' | 3 | 中等概率/影响 |
| 'LOW' | 2 | 低概率/低影响 |

##### 2. 修复 Include 对象 (影响 2 个测试)

```typescript
// 修复前
include: {
  task: {
    select: { id: true, title: true }
  }
}

// 修复后
include: {
  owner: {
    select: { id: true, name: true, email: true }
  }
}
```

##### 3. 添加缺失字段 (影响 4 个测试)

```typescript
// 新增字段
category: 'TECHNICAL',
riskLevel: 'CRITICAL',  // 由 probability × impact 计算
ownerId: 'user-123',
progress: 0,
contingency: null,
```

##### 4. 修复导入和 Mock (影响 2 个测试)

```typescript
// 添加缺失的导入
const { GET } = await import('@/app/api/v1/risks/route')
const { DELETE } = await import('@/app/api/v1/risks/[id]/route')

// 添加缺失的 Mock
vi.mocked(prisma.project.findUnique).mockResolvedValue(mockProject as any)
```

**修复成果**:

- ✅ 25 个测试全部通过 (100% 通过率)
- ✅ 无新失败引入
- ✅ 测试完全对齐实现行为
